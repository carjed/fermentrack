To restore from a previous backup of Fermentrack, select the backup file you have saved locally and choose "Restore from Backup" below.

NOTE - Restoring from a backup is primarily intended to be done to fresh installations of Fermentrack, as existing data will not be deleted from the database. If you restore from a backup into an installation of Fermentrack that is already in use, the two data sets will be merged - with data from the backup taking precedence over data that already exists in the database.

As restored data includes accounts, this may result in you being locked out of your installation of Fermentrack. If this happens, simply use the "docker-create-superuser.sh" script in fermentrack-tools to regain access.
